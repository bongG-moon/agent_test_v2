# 질문 가이드

이 문서는 어떤 질문을 하면 앱이 더 잘 이해하는지 설명합니다.  
사용자 입장에서 자연스럽게 질문하면서도, 시스템이 필요한 데이터와 조건을 놓치지 않도록 돕는 문서입니다.

## 1. 좋은 질문의 기본 원칙

아래 네 가지가 들어가면 대부분 잘 동작합니다.

- 언제
- 어디서
- 무엇을
- 어떻게 보고 싶은지

예:

```text
어제 DA공정에서 DDR5제품 생산 달성률 알려줘
```

이 질문에는 아래가 모두 들어 있습니다.

- 언제: `어제`
- 어디서: `DA공정`
- 무엇을: `DDR5제품`
- 어떻게: `생산 달성률`

## 2. 데이터 조회 질문 예시

- `오늘 DA공정 생산량 알려줘`
- `어제 WB공정 수율 보여줘`
- `오늘 DDR5제품 WIP 알려줘`
- `오늘 DA공정 설비 상태 알려줘`

## 3. 비교 질문 예시

- `오늘 생산과 목표를 같이 보여줘`
- `오늘 생산과 재공을 같이 보여줘`
- `어제 DA공정 생산량과 오늘 DA공정 생산량을 세부 공정별로 비교해줘`

## 4. 집계 질문 예시

- `오늘 DA공정에서 MODE별 생산량 알려줘`
- `오늘 DA공정에서 세부 공정별 생산량 비교해줘`
- `오늘 MODE별 평균 수율 보여줘`

## 5. 파생 지표 질문 예시

- `어제 DA공정 생산 달성률 알려줘`
- `오늘 DA공정 홀드 부하지수 알려줘`
- `오늘 DA공정 HOLD 이상여부 알려줘`

이런 질문은 단일 테이블이 아니라 여러 데이터셋이 함께 필요할 수 있습니다.  
앱은 기본 규칙과 사용자 등록 도메인을 보고 필요한 데이터셋을 함께 불러오려고 시도합니다.

## 6. 후속 분석 질문 예시

먼저 조회를 한 뒤:

1. `오늘 DA공정 생산량 알려줘`

그 다음 아래 같은 후속 질문을 할 수 있습니다.

- `MODE별로 묶어줘`
- `상위 5개만 보여줘`
- `오름차순으로 정렬해줘`
- `세부 공정 기준으로 다시 보여줘`

## 7. 도메인 등록 후 사용할 수 있는 질문 예시

### 데이터셋 표현 등록 후

등록:

```text
양품률이라는 표현은 수율 데이터셋을 뜻해.
```

질문:

```text
오늘 DA공정 양품률 알려줘
```

### 필터 별칭 등록 후

등록:

```text
후공정A는 D/A1, D/A2를 뜻해.
```

질문:

```text
오늘 후공정A 생산량 알려줘
```

### 계산 규칙 등록 후

등록:

```text
홀드 부하지수는 hold 테이블의 hold_qty 와 production 테이블의 production 을 같이 가져와서
hold_qty / production 으로 계산해줘.
```

질문:

```text
오늘 DA공정 홀드 부하지수 알려줘
```

## 8. 잘 안 풀릴 때 질문을 고치는 방법

### 너무 짧은 질문

```text
달성률 알려줘
```

이런 질문은 맥락이 부족할 수 있습니다.

더 좋은 질문:

```text
어제 DA공정에서 DDR5제품 생산 달성률 알려줘
```

### 비교 대상이 빠진 질문

```text
비교해줘
```

더 좋은 질문:

```text
어제 DA공정 생산량과 오늘 DA공정 생산량을 세부 공정별로 비교해줘
```

### 집계 기준이 빠진 질문

```text
생산량 정리해줘
```

더 좋은 질문:

```text
오늘 DA공정에서 MODE별 생산량 정리해줘
```

## 9. ENG'R 모드가 도움이 되는 상황

답이 이상하다고 느껴질 때 `ENG'R 모드`를 켜면 좋습니다.

이 모드에서는:

- 어떤 데이터셋이 사용됐는지
- 어떤 후처리 방식이 적용됐는지
- LLM 코드 생성인지, fallback인지

를 더 자세히 볼 수 있습니다.

## 10. 한 줄 요약

질문을 더 잘 이해시키는 가장 쉬운 방법은  
"날짜 + 공정 + 제품 조건 + 원하는 계산/집계 방식"을 한 문장 안에 함께 넣는 것입니다.
