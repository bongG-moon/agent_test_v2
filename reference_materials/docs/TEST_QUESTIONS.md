# Test Questions

이 문서는 현재 LangGraph 버전에서 주요 기능을 검증할 때 사용할 수 있는 질문 모음입니다.

질문을 그대로 복사해서 넣어도 되고, 비슷한 표현으로 바꿔도 됩니다.  
특히 이번 문서는 단순 질문 목록이 아니라 `체크리스트형 테스트 시나리오`로 정리했습니다.

## 1. 기본 기능 체크리스트

### 1-1. 생산 조회

- 질문: `오늘 DA공정 생산량 알려줘`
- 기대 결과:
  - `production` 조회가 실행됩니다.
  - DA 계열 공정만 보입니다.
  - 응답에 생산량 요약이 포함됩니다.

### 1-2. 재공 조회

- 질문: `오늘 WB공정 재공 얼마야?`
- 기대 결과:
  - `wip` 조회가 실행됩니다.
  - WB 계열 공정만 보입니다.
  - 재공 수량 중심 응답이 나옵니다.

### 1-3. 수율 조회

- 질문: `오늘 SAT공정 수율 알려줘`
- 기대 결과:
  - `yield` 조회가 실행됩니다.
  - 수율 관련 컬럼이 포함됩니다.

### 1-4. 설비 조회

- 질문: `오늘 FCB공정 설비 가동률 보여줘`
- 기대 결과:
  - `equipment` 조회가 실행됩니다.
  - 가동률 중심 응답이 나옵니다.

### 1-5. 홀드 조회

- 질문: `오늘 DA공정 홀드 현황 알려줘`
- 기대 결과:
  - `hold` 조회가 실행됩니다.
  - hold 수량이나 hold 사유가 포함됩니다.

### 1-6. HBM 제품 조회

- 질문: `오늘 HBM 제품 생산량 알려줘`
- 기대 결과:
  - HBM/3DS 의미가 `TSV` 제품 조건으로 해석됩니다.
  - TSV 계열 제품만 대상으로 생산 데이터가 조회됩니다.

### 1-7. Auto향 제품 조회

- 질문: `오늘 Auto향 제품 생산량 알려줘`
- 기대 결과:
  - Auto향 의미가 `MCP_NO` suffix 규칙으로 해석됩니다.
  - 해당 조건에 맞는 제품만 조회됩니다.

## 2. 후처리 / 연속 질문 체크리스트

### 2-1. 같은 표를 후처리하는 경우

1. 질문: `오늘 DA공정 생산량 알려줘`
2. 이어서 질문: `MODE별로 다시 보여줘`
3. 이어서 질문: `상위 3개만 보여줘`

기대 결과:

- 2번은 새 조회가 아니라 현재 표를 `MODE` 기준으로 그룹화합니다.
- 3번은 현재 표를 정렬/절단해서 상위 3개만 보여줍니다.

### 2-2. 첫 질문에서 바로 후처리하는 경우

- 질문: `오늘 DA공정에서 MODE별 생산량 알려줘`
- 기대 결과:
  - 첫 질문부터 조회 후 그룹화까지 한 번에 수행합니다.

### 2-3. FAMILY 컬럼 그룹화

- 질문: `오늘 DA공정에서 FAMILY별 생산량 알려줘`
- 기대 결과:
  - `FAMILY`가 실제 컬럼으로 인식됩니다.
  - `공정군` 같은 다른 의미로 잘못 해석되지 않아야 합니다.

### 2-4. 새 조회가 필요한 연속 질문

1. 질문: `오늘 DA공정 생산량 알려줘`
2. 이어서 질문: `WB공정 재공 얼마야?`

기대 결과:

- 1번은 `production` 조회
- 2번은 기존 생산 표를 재가공하지 않고 새로 `wip` 조회

## 3. 복수 조회 체크리스트

### 3-1. 생산 + 목표

- 질문: `오늘 생산량과 목표를 같이 보여줘`
- 기대 결과:
  - `production + target` 두 데이터셋이 함께 조회됩니다.

### 3-2. 생산 + 재공

- 질문: `오늘 DA공정에서 생산량과 재공을 같이 보여줘`
- 기대 결과:
  - `production + wip`가 함께 조회됩니다.

### 3-3. 생산 + 목표 + 재공

- 질문: `오늘 DA공정에서 생산량, 목표, 재공을 같이 보여줘`
- 기대 결과:
  - 3개 데이터셋이 함께 조회됩니다.
  - 공통 기준으로 병합된 표가 만들어집니다.

### 3-4. 날짜 비교

- 질문: `어제와 오늘 DA공정 생산량을 세부 공정별로 비교해줘`
- 기대 결과:
  - 같은 `production`을 날짜별로 2번 조회합니다.
  - 비교용 결과 표가 만들어집니다.

### 3-5. 날짜 비교 + 그룹화

- 질문: `어제 DA공정 생산량과 오늘 DA공정 생산량을 MODE별로 비교해줘`
- 기대 결과:
  - 날짜별 조회 후 `MODE` 기준 비교가 됩니다.

### 3-6. HBM 제품 + 복수 조회

- 질문: `오늘 HBM 제품 생산량과 재공 알려줘`
- 기대 결과:
  - `HBM/3DS -> TSV` 해석 후 `production + wip`가 함께 조회됩니다.

## 4. 파생 지표 / 계산 규칙 체크리스트

### 4-1. 달성률

- 질문: `어제 DA공정에서 DDR5제품 생산 달성률 알려줘`
- 기대 결과:
  - `production + target`이 자동 조회됩니다.
  - 달성률이 실제 계산됩니다.

### 4-2. 달성율 + 포화율

- 질문: `어제 일자 기준의 생산 달성율과 생산 포화율을 MODE/DEN/TECH/LEAD 그룹화해서 보여줘`
- 기대 결과:
  - `production + target + wip`가 필요에 맞게 조회됩니다.
  - 그룹별 계산 결과가 나옵니다.

### 4-3. FAMILY 포함 달성율 + 포화율

- 질문: `어제 일자 기준의 생산 달성율과 생산 포화율을 FAMILY/MODE/DEN/TECH/LEAD 그룹화해서 보여줘`
- 기대 결과:
  - `FAMILY`까지 포함해 그룹화됩니다.
  - `FAMILY`가 join key refinement에도 활용될 수 있습니다.

### 4-4. 홀드 부하지수

- 질문: `오늘 DA공정 홀드 부하지수 알려줘`
- 기대 결과:
  - `hold + production` 조회 후 비율 계산이 됩니다.

### 4-5. 설비재공 복합이상여부

- 질문: `오늘 DA공정 설비재공 복합이상여부 알려줘`
- 기대 결과:
  - `equipment + wip` 조회 후 조건형 판정이 됩니다.

## 5. 도메인 등록 체크리스트

### 5-1. 데이터셋 표현 등록

등록 문구:

```text
양품률이라는 표현은 수율 데이터셋을 뜻해.
```

테스트 질문:

- `오늘 DA공정 양품률 알려줘`

기대 결과:

- `yield` 조회로 연결됩니다.

### 5-2. 필터 별칭 등록

등록 문구:

```text
후공정A는 D/A1, D/A2를 뜻해.
```

테스트 질문:

- `오늘 후공정A 생산량 알려줘`

기대 결과:

- `D/A1`, `D/A2`로 확장되어 조회됩니다.

### 5-3. 계산 규칙 등록: 생산 목표 차이율

등록 문구:

```text
생산 목표 차이율은 production 테이블의 production 값과 target 테이블의 target 값을 같이 가져와서 계산해줘.
계산식은 (production - target) / target 이야.
결과 컬럼 이름은 plan_gap_rate 로 두고, 특별한 요청이 없으면 공정 기준으로 집계해서 보여줘.
```

테스트 질문:

- `어제 DA공정 생산 목표 차이율 알려줘`

기대 결과:

- `production + target`이 함께 조회됩니다.
- 계산 결과가 나옵니다.

### 5-4. 계산 규칙 등록: 홀드 부하지수

등록 문구:

```text
홀드 부하지수는 hold 테이블의 hold_qty 와 production 테이블의 production 을 같이 가져와서 계산해줘.
계산식은 hold_qty / production 이야.
결과 컬럼 이름은 hold_load_index 로 두고, 기본적으로 공정 기준으로 묶어서 보여줘.
```

테스트 질문:

- `오늘 DA공정 홀드 부하지수 알려줘`

기대 결과:

- `hold + production` 기반 계산이 됩니다.

### 5-5. 계산 규칙 등록: 설비재공 복합이상여부

등록 문구:

```text
설비재공 복합이상여부는 equipment 테이블의 가동률과 wip 테이블의 재공수량을 같이 보고 판단해줘.
가동률이 85 미만이면서 재공수량이 1800 이상이면 이상, 아니면 정상으로 판단해.
결과 컬럼 이름은 equipment_wip_abnormal_flag 로 두고, 기본 집계 기준은 공정으로 해줘.
```

테스트 질문:

- `오늘 DA공정 설비재공 복합이상여부 알려줘`

기대 결과:

- `equipment + wip` 기반 판정이 됩니다.

기대 결과:

## 6. 카디널리티 검증 체크리스트

### 6-1. 정상 병합되는 경우

- 질문: `어제 일자 기준의 생산 달성율과 생산 포화율을 MODE/DEN/TECH/LEAD 그룹화해서 보여줘`
- 기대 결과:
  - 정상 병합되어 결과가 나옵니다.

### 6-2. refinement가 필요한 경우

- 질문: `어제 일자 기준의 생산 달성율과 생산 포화율을 FAMILY/MODE/DEN/TECH/LEAD 그룹화해서 보여줘`
- 기대 결과:
  - `FAMILY`를 join key refinement에 활용해 정상 병합될 수 있습니다.

### 6-3. 기본적인 공통 key 병합

- 질문: `생산과 목표를 같이 보여줘`
- 기대 결과:
  - 공통 key가 충분하면 정상 병합됩니다.

### 6-4. 공통 기준이 불충분한 질문

- 질문: `생산과 목표를 공통 기준 없이 그냥 전부 섞어서 보여줘`
- 기대 결과:
  - 애매한 결합이면 안전하게 제한되거나 공통 기준 부족 안내가 나와야 합니다.

### 6-5. 다중 데이터셋 무리한 병합 요청

- 질문: `어제 생산, 목표, 재공을 아무 기준 없이 한 번에 다 합쳐서 보여줘`
- 기대 결과:
  - 공통 결합 기준이 부족하면 병합을 강행하지 않고 안내해야 합니다.

### 6-6. ENG'R 모드로 refinement 흔적 확인

- 질문: `어제 일자 기준의 생산 달성율과 생산 포화율을 FAMILY/MODE/DEN/TECH/LEAD 그룹화해서 보여줘`
- 기대 결과:
  - `ENG'R 모드`에서 merge note 쪽에 join key 확장이나 refinement 흔적이 보입니다.

### 6-7. 좁은 기준이 필요한 경우

- 질문: `오늘 DA공정 생산량과 목표를 MCP_NO 없이 MODE만 기준으로 비교해줘`
- 기대 결과:
  - 필요하면 더 좁은 key를 자동으로 찾습니다.
  - 못 찾으면 무리하게 병합하지 않고 안전하게 안내합니다.

## 7. mock 데이터 확장분 확인용 질문

지난번 mock 데이터 확장에서 `STANDARD / COMPUTE / MOBILE / HBM` 제품군과 `D/A3 + DDR5` 이상 신호를 넣어둔 상태를 확인하는 질문입니다.

- `D/A3`, `DDR5` 구간은 이상 신호 검증에 활용할 수 있습니다.

## 9. ENG'R 모드에서 같이 보면 좋은 것

- pandas 분석 방식이 무엇인지
- `llm_primary`, `retry`, `minimal_fallback` 중 어떤 경로였는지
- 복수 조회 시 어떤 join key를 사용했는지
- join refinement가 있었는지
