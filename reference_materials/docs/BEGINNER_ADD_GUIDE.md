# 초보자용 확장 가이드

이 문서는 Python 초보자 또는 비개발자가 "내가 원하는 의미를 이 앱에 어떻게 추가하는지"를 이해할 수 있도록 만든 문서입니다.

핵심은 아주 단순합니다.

- 코드 수정이 필요 없는 경우가 많습니다.
- 대부분은 `도메인 관리` 화면에서 줄글을 입력하면 됩니다.
- 정말 구조를 바꿔야 할 때만 코드를 수정하면 됩니다.

## 1. 먼저 구분해야 할 것

새 요구사항이 들어오면 가장 먼저 아래 둘 중 무엇인지 구분하세요.

### A. 도메인 관리 화면으로 해결 가능한 경우

이 경우는 보통 코드를 직접 수정할 필요가 없습니다.

예:

- 새로운 현장 표현을 추가하고 싶다
- 공정 별칭을 추가하고 싶다
- 계산 규칙을 추가하고 싶다
- 이상/정상 판정 규칙을 추가하고 싶다

### B. 개발자가 코드까지 수정해야 하는 경우

예:

- 아예 새로운 데이터셋을 추가하고 싶다
- mock 데이터 구조를 바꾸고 싶다
- 기본 공정 그룹을 코드 레벨에서 바꾸고 싶다
- 기본 내장 규칙을 수정하고 싶다

## 2. 일반 사용자는 보통 도메인 관리 화면만 쓰면 된다

추천 순서는 아래와 같습니다.

1. `도메인 관리`로 이동합니다.
2. 줄글 설명을 입력합니다.
3. `미리보기`를 눌러 구조화 결과를 확인합니다.
4. 경고/오류를 읽습니다.
5. 문제 없으면 `등록`합니다.
6. `채팅 분석` 화면으로 돌아가 실제 질문으로 테스트합니다.

## 3. 어떤 문장을 넣으면 좋은가

문장은 짧고 명확할수록 좋습니다.  
아래 네 가지 패턴으로 생각하면 쉽습니다.

### 1. 데이터셋 표현 등록

```text
양품률이라는 표현은 수율 데이터셋을 뜻해.
```

이런 문장은:

- 사용자가 `양품률`이라고 질문했을 때
- 앱이 `yield` 데이터셋을 선택하도록 돕습니다.

### 2. 필터 별칭 등록

```text
후공정A는 D/A1, D/A2를 뜻해.
```

이런 문장은:

- 사용자가 `후공정A`라고 질문했을 때
- 실제 필터를 `D/A1`, `D/A2`로 확장하게 합니다.

### 3. 계산 규칙 등록

```text
홀드 부하지수는 hold 테이블의 hold_qty 와 production 테이블의 production 을 같이 가져와서
hold_qty / production 으로 계산해줘.
```

이런 문장은:

- 필요한 데이터셋이 `hold`, `production`이라는 점
- 계산식이 `hold_qty / production`이라는 점
- 이후 pandas 코드 생성 시 어떤 방향으로 가야 하는지를 알려줍니다.

### 4. 판정 규칙 등록

```text
장비 가동 이상여부는 equipment 테이블의 가동률이 85 미만이면 이상,
아니면 정상으로 판단해줘.
```

이런 문장은:

- 어떤 데이터셋과 컬럼을 써야 하는지
- 어떤 조건이면 `이상`인지
- 어떤 결과 컬럼을 만들어야 하는지를 시스템이 이해하게 돕습니다.

## 4. 좋은 문장을 만드는 팁

가능하면 아래 정보를 함께 넣어 주세요.

- 어떤 데이터셋을 쓰는지
- 어떤 컬럼을 쓰는지
- 계산식이 무엇인지
- 조건식이 무엇인지
- 이상/정상 같은 결과 표현이 무엇인지
- 기본 집계 기준이 무엇인지

예를 들어 좋은 문장은 이런 식입니다.

```text
생산 목표 차이율은 production 테이블의 production 값과 target 테이블의 target 값을 가져와서
(production - target) / target 으로 계산해줘.
기본 집계 기준은 공정으로 해줘.
결과 컬럼 이름은 plan_gap_rate 로 해줘.
```

## 5. 미리보기에서 무엇을 봐야 하나

미리보기 결과에서는 아래를 순서대로 확인하세요.

### 제목

문장이 어떤 주제로 이해되었는지 봅니다.

### 데이터셋 키워드

내가 입력한 문장이 특정 데이터셋 표현으로 이해되었는지 확인합니다.

### 필터 값 그룹

별칭이 실제 값 목록으로 잘 풀렸는지 확인합니다.

### 분석 규칙

계산/판정 로직이 아래처럼 잘 들어갔는지 봅니다.

- `required_datasets`
- `required_columns`
- `source_columns`
- `calculation_mode`
- `condition`
- `decision_rule`
- `output_column`

### 경고/오류

특히 이 부분을 반드시 읽어야 합니다.

## 6. 경고와 오류는 어떻게 이해하면 되나

### warning

대체로 저장은 가능하지만, 의미가 기존 것과 겹치거나 애매할 수 있다는 뜻입니다.

예:

- 이미 같은 데이터셋에 비슷한 keyword가 있음
- 계산 규칙인데 결과 컬럼 이름이 없음
- condition 기반 규칙인데 condition 문장이 너무 빈약함

### error

실제 충돌이 있어 저장하면 안 되는 경우입니다.

예:

- `수율`을 생산 데이터셋 뜻으로 새로 등록하려는 경우
- 기존 규칙과 다른 의미인데 같은 이름/별칭을 쓰려는 경우
- unsafe text가 들어간 경우

## 7. 등록 후 어떻게 테스트하나

### 예시 1. 데이터셋 표현 등록 테스트

등록:

```text
양품률이라는 표현은 수율 데이터셋을 뜻해.
```

질문:

```text
오늘 DA공정 양품률 알려줘
```

기대 결과:

- `yield` 데이터셋이 선택되어야 합니다.

### 예시 2. 필터 별칭 등록 테스트

등록:

```text
후공정A는 D/A1, D/A2를 뜻해.
```

질문:

```text
오늘 후공정A 생산량 알려줘
```

기대 결과:

- `process_name`이 `D/A1`, `D/A2`로 확장되어야 합니다.

### 예시 3. 계산 규칙 등록 테스트

등록:

```text
생산 달성율은 production 테이블의 production 값과 target 테이블의 target 값을 가져와서
production / target 으로 계산해줘.
```

질문:

```text
어제 DA공정 생산 달성율 알려줘
```

기대 결과:

- `production`, `target`이 같이 조회되어야 합니다.
- 최종 답변에서 실제 달성율 계산이 나와야 합니다.

### 예시 4. 판정 규칙 등록 테스트

등록:

```text
HOLD 이상여부는 wip 테이블의 상태 컬럼이 HOLD, REWORK, WAIT_MATERIAL 중 하나이면 이상,
아니면 정상으로 판단해줘.
```

질문:

```text
오늘 DA공정 HOLD 이상여부 알려줘
```

기대 결과:

- `wip` 데이터셋이 조회되어야 합니다.
- 상태 기준 판정 결과가 포함되어야 합니다.

## 8. 등록한 도메인을 삭제하고 싶으면

가능합니다.

1. `도메인 관리 > 목록`으로 이동합니다.
2. 지우고 싶은 엔트리를 찾습니다.
3. `삭제` 버튼을 누릅니다.

이 기능은 다음 상황에서 특히 유용합니다.

- 테스트용 규칙을 잠깐 넣어봤을 때
- 문장을 잘못 입력했을 때
- 같은 의미를 더 좋은 설명으로 다시 등록하고 싶을 때

## 9. 그래도 안 되면 어디를 봐야 하나

일반 사용자 관점에서는 아래 순서로 확인하는 것이 좋습니다.

1. 입력 문장이 너무 짧거나 애매하지 않았는지
2. 미리보기 결과가 의도대로 나왔는지
3. 경고/오류가 있었는지
4. 등록 후 실제 질문이 규칙 이름이나 별칭을 포함하고 있는지
5. `ENG'R 모드`에서 어떤 데이터셋과 분석 방식이 사용됐는지

## 10. 한 줄 조언

처음부터 완벽한 문장을 쓰려고 하기보다,  
"어떤 데이터셋", "어떤 컬럼", "어떤 계산", "어떤 조건" 이 네 가지를 분명하게 적는 습관을 들이면 거의 대부분 잘 동작합니다.
